## ⑩ 蒙特卡洛路径模拟（Monte Carlo）

### 为什么需要？
现有系统预测单场比赛准确率不错，但**多场比赛联动时**（如争冠/保级/出线形势），市场往往会定价偏差。比如"某队不仅要赢还要另一场平局"的复杂条件，人类很难算清楚，但蒙特卡洛可以。

### 核心原理

```
Step 1: 用Dixon-Coles/Skellam生成每场的比分概率分布
Step 2: 从分布中随机抽取1个比分（1场模拟）
Step 3: 重复到所有场次（1轮完整模拟）
Step 4: 计算"结果条件"（如某队是否出线）
Step 5: 重复10万次 → 统计累计次数/10万 = 概率
```

### 在竞彩中的应用

**应用1：14场/任选9的"胆材保险"计算**
```
传统做法：凭感觉选胆材
蒙特卡洛做法：
  - 对14场各跑1万次模拟
  - 计算每场的"胆材可靠性" = 主胜概率 × (1 - 冷门方差)
  - 选出可靠性最高的5场做胆材
  - 这比凭感觉选胆材科学得多
```

**应用2：组合串关的"真实胜率"评估**
```
传统做法：SP相乘就当胜率
蒙特卡洛做法：
  - 对串关内的N场比赛跑10万次模拟
  - 计算"串关全中"的真实概率
  - 对比SP乘积的隐含概率 → 找出价值偏差
  - 公式：真实胜率 = 全中次数 / 10万
```

**应用3：EV校准**
```
如果蒙特卡洛算出的真实概率 > 市场概率(1/SP) → 有价值投注
例如：3串1 SP=8.0 → 市场认为中奖概率=12.5%
      蒙特卡洛算出真实概率=15% → EV = (0.15×8 - 1) = +20% ✅
```

### 如何落地（纯Python，不需要额外库）

```python
import random, math
from collections import Counter

# Step 1: 泊松模拟一场比赛
def simulate_match(lambda_home, lambda_away):
    home_goals = random.poisson(lambda_home)  # 或用负二项更精确
    away_goals = random.poisson(lambda_away)
    return home_goals, away_goals

# Step 2: 模拟多场比赛的串关
def simulate_parlay(matches, n_sim=100000):
    """matches = [(λ_home, λ_away, required_result), ...]
       required_result: 'H'主胜, 'D'平, 'A'客胜
    """
    wins = 0
    for _ in range(n_sim):
        all_win = True
        for lam_h, lam_a, req in matches:
            h, a = simulate_match(lam_h, lam_a)
            result = 'H' if h > a else ('D' if h == a else 'A')
            if result != req:
                all_win = False
                break
        if all_win:
            wins += 1
    return wins / n_sim  # 真实中奖概率

# Step 3: 对比市场赔率
market_prob = 1 / 8.0  # SP=8.0
true_prob = simulate_parlay([(1.5, 0.8, 'H'), (2.1, 0.6, 'H')])
ev = (true_prob * 8.0 - 1) * 100
print(f"市场概率: {market_prob*100:.1f}%")
print(f"真实概率: {true_prob*100:.1f}%")
print(f"EV: {ev:+.1f}%")
```

### 与现有体系结合
- **复用**：Dixon-Coles的λ1/λ2已经算好了，直接喂给蒙特卡洛
- **输出**：蒙特卡洛概率作为"第10个模型"加入BMA融合层
- **场景**：14场胆材选择、任9方案优化、串关价值评估

## ⑪ 堆叠集成（Stacking Ensemble）

### 为什么需要？
现有BMA是**固定权重**的模型平均。堆叠集成用**机器学习**学习最优权重——让数据自己决定哪些模型什么时候更可信。

### BMA vs Stacking

| | BMA（现有） | Stacking（新增） |
|:--|:-----------|:-----------------|
| 权重 | 固定(0.30/0.22/0.15...) | **数据驱动动态权重** |
| 覆盖 | 仅5个概率模型 | +特征向量，+场景因素 |
| 学习方式 | 人手调参 | **交叉验证自动训练** |
| 适应性 | 调参慢 | 每20场复盘后自动重训练 |

### 核心原理

```
层次0（Level-0）：多个基模型
  ├── M1 Dixon-Coles → 输出P1(主/平/客)
  ├── M2 Skellam → 输出P2(主/平/客)  
  ├── M3 有序Probit → 输出P3(主/平/客)
  ├── M4 市场赔率反推 → 输出P4(主/平/客)
  ├── M5 联赛基准率 → 输出P5(主/平/客)
  ├── M6 行为反转因子 → 输出P6(主/平/客) [新增]
  └── M7 蒙特卡洛 → 输出P7(主/平/客) [新增]

层次1（Level-1）：元模型
  输入：P1-P7（共21个概率值）+ 场景特征（联赛/赔率区间/信息完备度）
  输出：最终概率P_final(主/平/客)
  算法：正则化逻辑回归(ElasticNet) 或 轻量级XGBoost
  训练：5折交叉验证 → 用实际赛果(0/1)做标签
```

### 如何落地

```python
# 伪代码：堆叠集成训练流程
from sklearn.linear_model import LogisticRegression
from sklearn.model_selection import KFold

# Step 1: 准备训练数据
# 每行 = 1场比赛，特征 = 7个模型的21个概率值
# 标签 = 实际结果(主胜=1/平局=0/客胜=0)
X = []  # 特征矩阵 (N场比赛 × 21个模型概率)
y = []  # 标签 (N场比赛 × 1)

# Step 2: 5折交叉验证
kf = KFold(n_splits=5, shuffle=True)
meta_features = []

for train_idx, val_idx in kf.split(X):
    # 在训练集上训练元模型
    meta_model = LogisticRegression(penalty='elasticnet', solver='saga')
    meta_model.fit(X[train_idx], y[train_idx])
    # 在验证集上生成元特征
    val_pred = meta_model.predict_proba(X[val_idx])
    meta_features.append(val_pred)

# Step 3: 用全部数据训练最终元模型
final_model = LogisticRegression()
final_model.fit(np.vstack(meta_features), y)

# Step 4: 预测时
def predict(home_lam, away_lam, league, odds):
    # 各基模型算概率
    p1 = dixon_coles(home_lam, away_lam)  # M1
    p2 = skellam(home_lam, away_lam)      # M2
    # ... 其他模型
    # 堆叠融合
    features = np.concatenate([p1, p2, p3, p4, p5, p6, p7])
    return final_model.predict_proba([features])[0]
```

### 训练数据从哪里来
- **复盘数据**：每场比赛的模型概率CV2（交叉验证预测值）记录到C50-C54列
- 累计20场以上就可以初训
- 100场以上效果显著
- 初始阶段可以先用BMA，等数据够了再切Stacking

### 与现有体系结合
- **替代BMA融合层**：不冲突，初期BMA兜底，Stacking做备选
- **复盘编译器自动训练**：每次有新的复盘数据就增量训练
- **权重可视化**：元模型的系数直接告诉每个模型在不同场景下的重要性

## ⑫ 行为金融反转策略（Behavioral Finance）

### 为什么需要？
人不是完全理性的。**市场参与者会犯系统性的错误**，这些错误会体现在赔率上，形成价值投注机会。

### 识别的5种行为偏差

| 偏差类型 | 表现 | 利用方法 |
|:---------|:-----|:---------|
| 🔥 **近期偏差** | 最近赢的队被高估，最近输的队被低估 | 连胜队反向看，连败队找价值 |
| 📰 **媒体热度** | 上头条多的队赔率被压低 | 热度高位→投注对方 |
| 🐑 **从众效应** | 支持率高的方向赔率偏低 | 支持率>70%时反向思考 |
| 😱 **后悔厌恶** | 不敢买冷门（怕错了后悔） | 冷门赔率常有溢价 |
| 📊 **锚定效应** | 开盘赔率影响后续判断 | 对比开盘与当前赔率差异 |

### 行为因子计算

```
# 近期偏差因子
momentum_bias = (近5场积分率 - 赛季积分率) × 100
# 正值 = 市场偏高估该队，负值 = 偏低估

# 媒体热度因子（简易版）
# 用赔率变化幅度代理：有重大新闻的队赔率波动大
volatility = |当前主SP - 早盘主SP| / 早盘主SP
# >5% 说明有信息冲击，可能有偏差

# 从众效应因子
# 用竞彩官网的"支持率"数据（如首页有）
crowd_bias = 主队支持率% - 主队真实概率%
# 支持率 > 真实概率 → 过度追捧
```

### 反转策略规则

```
基础规则：
  IF 近期偏差因子 > 15%（市场高估主队）
     AND 模型概率 < 市场概率
     → 反向投注客队

  IF 从众偏差 > 10%（投注者一边倒）
     AND 真实概率与市场概率差 > 5%
     → 投注被低估的一方

  IF 媒体热度高（赔率波动>10%）
     → 等待波动平息，投注"非新闻方"
```

### 与现有体系结合
- **新增第⑨层数据源**：行为偏差作为一个独立视角
- **与Time Decay互补**：时间衰减给近期数据更高权重，行为反转利用这种过度加权
- **作为"反向指标"**：当所有模型都看好A时，如果行为因子也过热→减仓

## ⑬ 日历效应（Calendar Effects）

### 为什么需要？
最简单的特征往往最有效。周几比赛、休息天数、赛季阶段这些**一个字段就能加进去**的变量，测试后发现效果显著。

### 已有的发现（25,168场数据验证）

| 日历因子 | 影响 | 已用？ |
|:---------|:-----|:------:|
| 周中vs周末 | 周中比赛主胜率下降约8% | ❌ 未用 |
| 国际比赛日后 | 强队主胜率下降12% | ❌ 未用 |
| 赛季末(最后5轮) | 无欲无求队爆冷率上升 | ❌ 未用 |
| 长假前后 | 球员注意力不集中 | ❌ 未用 |
| 首个主场vs连续主场 | 连续主场优势递减 | ❌ 未用 |
| 休息天数差 | 每多休息1天，主胜+2-3% | 已用(时序层) |

### 新增日历特征

```python
# 每日自动生成的特征

    'is_weekend': 1 if 周5/6/7 else 0,
    'days_since_last_match': 主队距上场天数 - 客队距上场天数,
    'is_post_international': 1 if 国际比赛日后3天内 else 0,
    'season_phase': '开始' if 轮次<=5 else '中期' if 轮次<=25 else '末期',
    'is_motivation_gap': 1 if 保级vs无欲无求 else 0,  # 战意差
    'is_home_streak': 连续主场场数,  # 连续主场>2 → 优势递减
    'is_derby': 1 if 德比战 else 0,
}
```

### 与现有体系结合
- **直接新增到15维特征**的F16-F22（7个日历特征）
- **初始权重设为0.02-0.03**（因为是新特征，等验证后再加）
- **独立回测**：先只用日历特征做预测，看是否比随机好
