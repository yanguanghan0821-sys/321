## 一、Prompt-Optimizer 是什么

### 核心原理

Prompt-Optimizer 是一个自动优化提示词的 Python 库。它的工作逻辑：

```
输入: 基础提示词 + 目标输出样例
  ↓
① 在提示词空间内生成变体（换词、重排结构、改语气）
② 每个变体发起一次LLM调用
③ 用预设指标（余弦相似度/BLEU/ROUGE）评估输出质量
④ 选出最优变体
  ↓
输出: 优化后的提示词
```

**核心思想**：不是人脑去猜"怎么改提示更好"，而是让算法自动试、自动选。

### 支持的功能

| 特性 | 说明 |
|:-----|:------|
| 优化算法 | 暴力搜索(穷举)、贝叶斯优化(高效)、随机搜索 |
| 评估指标 | 余弦相似度、BLEU、ROUGE、自定义metric函数 |
| 使用方式 | `pip install prompt-optimizer` → Python调用 |

## 二、应用到竞彩分析——自优化提示系统

### 我们的"提示词"在哪里？

我们的情况比较特殊——提示词不是一句两句，而是**固化成skill的长篇方法论**：

| 传统用法 | 我们的用法 |
|:---------|:-----------|
| 一句prompt "分析这场比赛" | 整个`football-betting-jingcai`技能(~200KB) |
| 改几个词看效果 | 改SOP流程、调权重参数 |
| 优化方向 | 优化分析步骤 → 提高命中率 |

### 等价实现：复盘驱动自优化循环

我们不需要外部库——因为**复盘本身就是prompt-optimizer**！

```
                 复盘(我们的"评估函数")
                       ↑
① 初始提示(技能) → ② 执行分析 → ③ 输出推荐 → ④ 实际结果
                                              ↓
                   ⑤ 偏差分析 → ⑥ 修正技能/权重 → 回到①
```

### 实战：引用prompt-optimizer思想优化6步SOP

借鉴其**变体-评估-选择**循环，我们对每个步骤做优化：

#### Step① 查截止时间 —— 变体试验

| 变体 | 方法 | 评估结果 |
|:-----|:-----|:---------|
| A | 只查sporttery.cn公告 | ❌ 有漏查风险 |
| B | sporttery.cn + 详细表格 | 🟡 两源交叉 |
| C | **三源交叉(sporttery+表格+500.com) ✅采用** | ✅ 最可靠 |

#### Step③ 9层分析 —— 变体试验

| 变体 | 层数 | 评估结果 |
|:-----|:----:|:---------|
| v1.0 | 5层 | 🟡 够用但不够全面 |
| v2.0 | **9层 ✅采用** | ✅ 更全面，系统风险/时序/外部/叙事都很实用 |

#### Step④ 复式推荐 —— 变体试验

| 变体 | 格式 | 评估结果 |
|:-----|:-----|:---------|
| A | 单场多玩法(比分+半全场+让球) | ❌ 用户说不对 |
| B | **串关内每场多选(双选/三选) ✅采用** | ✅ 用户认可 |

**这就是prompt-optimizer的思想：不靠猜，靠试错+反馈来逼近最优方案。**

## 三、每日自优化机制

### 自动追踪的优化指标

```
Prompt质量评分 = 
    0.30 × 命中率(实际) 
  + 0.20 × 覆盖度(是否所有场次都覆盖)
  + 0.20 × 完整度(评分卡是否完整)
  + 0.15 × 时效(推送是否准时)
  + 0.15 × 用户满意度(是否被纠正)
```

每项0-10分，每日自动评分。

### 版本进化表

```
v1.0 (2026-05-05) → 基础三层框架
v2.0 (2026-05-05) → 加铁律+凯利
v3.0 (2026-05-05) → 信息驱动+联赛校正
v3.2 (2026-05-06) → 复式推荐格式修正+数据采集优化
v4.0 (2026-05-06) → 五层决策体系(Dixon-Coles/Skellam/BMA)
v4.1 (2026-05-06) → v2.0 九层体系+6步零缺陷SOP+复盘风险对照
```

每次修正都是prompt-optimizer思想的体现：**发现问题 → 分析根因 → 修改技能 → 下次验证 → 评估效果 → 继续迭代**。

## 四、延伸：DSPy (⭐20k) 的启示

DSPy比prompt-optimizer更强大。它的核心概念照搬到我们的体系：

| DSPy概念 | 我们的映射 | 说明 |
|:---------|:-----------|:-----|
| `Signature`（输入/输出签名） | 评分卡模板 | 定义每场分析必须输出什么 |
| `Module`（模块） | 9层分析各层 | 每层是一个独立分析模块 |
| `Compiler`（编译器） | 复盘偏差分析 | 用实际结果反向优化各层参数 |
| `Metric`（评估指标） | 命中率+覆盖度+完整度 | 定义什么样的分析算"好" |
| `TrainSet`（训练集） | 历史复盘数据(27条) | 积累越多，优化越准 |

### 未来方向：数据驱动参数自动调优

当积累到**100条复盘数据**后（约1个月），可以实现：
```
1. 统计各层偏差分布 → 找出最常出错的层
2. 统计各特征权重的实际预测贡献 → 自动调整权重
3. 统计BMA各模型的实际准确率 → 调整模型权重
```

到那时就不需要手动调参了——**数据自己告诉你该怎么调**。
