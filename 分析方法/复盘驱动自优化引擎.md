# ⚡ 复盘驱动自优化引擎 — 竞彩版的DSPy编译器

> 融合 prompt-optimizer 搜索思想 + DSPy(⭐20k) 编译器模式
> 不装库也能跑——用复盘数据当训练集，命中率当打分指标

---

## 一、核心架构

```
DSPy概念 → 我们的映射 → 每日执行
──────────────────────────────────
Signature → 分析评分卡模板 → 每场分析必须输出方向+比分+半全场+理由
Module    → 6步SOP流程    → 查截止→采数据→9层分析→出推荐→推送→复盘
Metric    → 命中率+综合评分 → 每日复盘计算
Compiler  → 复盘偏差分析   → 定位错误层→生成修正→模拟验证→部署上线
TrainSet  → 历史复盘记录  → 主表记录的实际比分和预测方向
```

---

## 二、Compiler核心逻辑（每日自动执行）

### 算法流程图

```
每天复盘完成后：
  ① 收集昨日所有比赛实际结果
  ② 计算Metric分（命中率+覆盖度+完整度）
     ↓
  ③ 分数下降 或 连续5天不提升？
     ├─ YES → 触发自优化
     │     ↓
     │  ④ 在历史数据上找出最近10次失败案例
     │  ⑤ LLM分析失败根因 → 生成修正建议
     │  ⑥ 用最近20场历史数据模拟新方案
     │     ↓
     │  ⑦ 模拟分数 > 当前分数？
     │     ├─ YES → 部署新方案（更新技能/权重）
     │     └─ NO  → 保留当前方案+记录"建议被拒"
     │
     └─ NO → 保持当前配置
```

### Metric评分公式

```
综合评分 = 
    0.40 × 命中率(最近20场)
  + 0.20 × 覆盖度(所有场次都分析了没)
  + 0.15 × 完整度(评分卡全不全)  
  + 0.15 × 时效(推送是否准时)
  + 0.10 × 用户反馈(被纠正的频次)

每项归一化到[0,1]，综合评分[0,1]
触发阈值：连续5天不提升 或 单日下降>0.1
```

## 三、失败回放 → 修正建议 → 模拟验证

### 步骤①：失败回放

```
从C38(命中列)抓最近10条 ❌未中 记录：

| 场次 | 预测方向 | 实际结果 | 偏差层(C45) | 根因(C46) |
|:----:|:--------:|:--------:|:----------:|:----------|
| XXX  | 🟢主胜    | 🔴客胜    | ②信息采集  | 遗漏了核心射手X的伤病 |
| XXX  | 🟢主胜    | 🟡平局    | ③模型预测  | Dixon-Coles低估了平局概率 |
```

### 步骤②：根因聚合

```
错误统计（从C46根因列聚合）：
  市场解读错误: N次 → 看是不是经常被诱导信号骗了
  信息遗漏: N次 → 查是哪个信息源经常出问题
  模型偏差: N次 → 查M1/M2/M3哪个模型最常不准
  纪律违反: N次 → 又是没遵守选场铁律？
```

### 步骤③：生成修正建议

```
根据根因聚合结果，自动生成不同的修正方向：

| 触发条件 | 修正建议 |
|:---------|:---------|
| 市场解读错误>3次 | 提高竞彩SP偏离度阈值 或 增加"三向一致"验证 |
| 信息遗漏>3次 | 增加一个信息采集步骤 或 更换信息源 |
| 某模型偏差>3次 | 降低该模型BMA权重0.02，分散给其他模型 |
| 纪律违反>3次 | 强化铁律检查清单 或 加自动仓位校验 |
```

### 步骤④：模拟验证

```
用过去20场历史数据做"模拟分析"：
  1. 用新方案重新分析过去20场比赛
  2. 计算新方案的"理论命中率"
  3. 与旧方案的实际命中率对比
  4. 新 > 旧？→ 部署新方案
  5. 新 <= 旧？→ 保留旧方案，记日志
```

---

## 四、每日执行脚本

```
复盘完成后：
Step 1: 读C38(命中列)，算最近20场命中率
Step 2: 若命中率下降 或 连续5天flat → 触发编译器
Step 3: 读C45-C46(偏差层+根因)，统计分布
Step 4: 用 LLM 生成修正建议
Step 5: 用历史数据模拟验证
Step 6: 通过 → 更新技能/权重
Step 7: 写入修正日志
```
